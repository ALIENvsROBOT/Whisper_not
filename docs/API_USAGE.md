# API Usage

## Authentication and base URL

```text
Base URL: http://HOST:9000/v1
Header: Authorization: Bearer <WHISPER_API_KEY>
```

The primary endpoint is:

```text
POST /v1/audio/transcriptions
```

Requests use `multipart/form-data`.

## Request fields

| Field | Type | Default | Description |
|---|---|---|---|
| `file` | file | required | Audio or video input supported by PyAV |
| `model` | string | `whisper-1` | Compatibility alias; the startup model performs inference |
| `language` | string | auto | ISO-639-1 code or `auto` |
| `prompt` | string | empty | Initial transcription prompt |
| `response_format` | string | `json` | `json`, `text`, `verbose_json`, `diarized_json`, `srt`, or `vtt` |
| `temperature` | number | `0` | Decoding temperature |
| `stream` | boolean | false | Stream text deltas using SSE |
| `timestamp_granularities[]` | string[] | segment | Include `word` for word timestamps |
| `diarize` | boolean | false | Enable speaker diarization for this request |
| `num_speakers` | integer | auto | Exact positive speaker count |
| `diarize_threshold` | number | `0.5` | Auto-clustering threshold |
| `download` | boolean | false | Add an attachment filename |

## Default Open WebUI request

```bash
curl http://localhost:9000/v1/audio/transcriptions \
  -H "Authorization: Bearer $WHISPER_API_KEY" \
  -F file=@voice.webm \
  -F model=whisper-1
```

```json
{"text":"Transcribed text"}
```

This path does not run word timestamp generation or speaker diarization.

## Verbose JSON

```bash
curl http://localhost:9000/v1/audio/transcriptions \
  -H "Authorization: Bearer $WHISPER_API_KEY" \
  -F file=@meeting.mp3 \
  -F model=whisper-1 \
  -F response_format=verbose_json
```

The response includes language detection, duration, and timestamped segments.

## Word timestamps

```bash
curl http://localhost:9000/v1/audio/transcriptions \
  -H "Authorization: Bearer $WHISPER_API_KEY" \
  -F file=@meeting.mp3 \
  -F model=whisper-1 \
  -F response_format=verbose_json \
  -F "timestamp_granularities[]=word"
```

Word timestamps are generated by faster-whisper only when requested.

## Speaker diarization

```bash
curl http://localhost:9000/v1/audio/transcriptions \
  -H "Authorization: Bearer $WHISPER_API_KEY" \
  -F file=@meeting.mp3 \
  -F model=whisper-1 \
  -F response_format=verbose_json \
  -F diarize=true
```

Each segment receives a `speaker` field such as `SPEAKER_00`. The result is
not limited to two speakers; automatic clustering can emit
`SPEAKER_02`, `SPEAKER_03`, and additional labels as needed.

When the exact speaker count is known:

```bash
-F num_speakers=4
```

Leave `num_speakers` unset for automatic detection. In automatic mode,
`diarize_threshold` controls clustering: lower values generally produce more
speaker clusters, while higher values produce fewer.

## OpenAI-style diarized JSON

Either `response_format=diarized_json` or
`model=gpt-4o-transcribe-diarize` activates local diarization:

```bash
curl http://localhost:9000/v1/audio/transcriptions \
  -H "Authorization: Bearer $WHISPER_API_KEY" \
  -F file=@meeting.mp3 \
  -F model=gpt-4o-transcribe-diarize \
  -F response_format=diarized_json
```

```json
{
  "task": "transcribe",
  "duration": 12.3,
  "text": "A: Hello.\nB: Hi.",
  "segments": [
    {
      "type": "transcript.text.segment",
      "id": "seg_001",
      "start": 0.0,
      "end": 1.2,
      "text": "Hello.",
      "speaker": "A"
    }
  ],
  "usage": {
    "type": "duration",
    "seconds": 12
  }
}
```

Known-speaker reference audio and token log probabilities are not supported.

## Subtitle and text output

SRT:

```bash
-F response_format=srt
```

WebVTT:

```bash
-F response_format=vtt
```

Plain text:

```bash
-F response_format=text
```

When diarization is enabled, text and subtitle formats include speaker labels.

## Downloaded response

Add:

```bash
-F download=true
```

The server adds a safe `Content-Disposition` attachment filename derived from
the uploaded filename. It does not persist the generated response.

## Streaming

```bash
curl -N http://localhost:9000/v1/audio/transcriptions \
  -H "Authorization: Bearer $WHISPER_API_KEY" \
  -F file=@meeting.mp3 \
  -F model=whisper-1 \
  -F stream=true
```

Streaming returns OpenAI-style SSE delta and done events. Streaming cannot be
combined with diarization, word timestamps, or downloads.

## Translation

```text
POST /v1/audio/translations
```

The translation endpoint converts supported source audio to English and
supports the common non-diarization output formats.

## Models endpoint

```text
GET /v1/models
```

It exposes:

- `whisper-1`;
- the configured startup model;
- `gpt-4o-transcribe-diarize`.

The startup model is always the model that performs inference.

## Error behavior

- `400`: invalid or unsupported request combination.
- `401`: missing or invalid bearer token.
- `413`: upload exceeds `WHISPER_MAX_UPLOAD_MB`.
- `429`: the bounded transcription queue is full or the queue wait timed out.
- `500`: transcription or diarization execution failed.
- `503`: startup model is not ready.
